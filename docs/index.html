Memory Model · AIEL Phase 4

Four Types of Personal Memory

Every piece of knowledge is classified, embedded (all-MiniLM-L6-v2, 384-dim), and stored in PostgreSQL with pgvector. Memories strengthen on access (reinforce-on-read) and decay when unused — modelled after how human memory actually works.

Architecture · Hexagonal / Ports & Adapters

Module Structure

Port interfaces live in core-domain (no Spring dependency). Adapters in core-memory. The Spring Boot application in core-api wires everything together. Cross-module calls only via port interfaces — never direct imports.

core-domain
No Spring dependency

Domain Types + Port Interfaces

Records: PersonalMemory, MemoryType, CognitiveSession, PersonalContext, GdprConsent. Port interfaces: PersonalMemoryStore, PersonalContextProvider, CognitiveSessionStore, GdprConsentStore.

core-memory
Spring JDBC adapters

pgvector Store + Embedding + Decay

PGVectorPersonalMemoryStore — cosine similarity search, upsert, reinforce-on-read (+0.1 strength per retrieval). PersonalEmbeddingService — Ollama all-MiniLM-L6-v2, 384-dim, graceful fallback via @ConditionalOnProperty. MemoryDecayService — daily decay at 02:00 UTC, weekly purge at 03:00 UTC Sunday. GridFeedbackConsumer — optional Kafka listener.

core-api
Port 8082

Spring Boot Application

Controllers: PersonalContextController (Grid integration), PersonalMemoryController, CognitiveSessionController, GdprController. Flyway V001–V004 migrations. Actuator health/readiness/liveness probes. CoreApiConfig wires all port implementations.

core-infra
Infrastructure

Docker + Kubernetes

Multi-stage Dockerfile (eclipse-temurin:21-jre-noble, non-root uid 1000). Docker Compose (postgres-core on 5433, aether-core on 8082). Kubernetes: namespace.yaml, deployment.yaml (2 replicas, HPA 2–8), service.yaml (ClusterIP + ConfigMap + Secret template).

API Contract · AIEL Phase 6

REST Endpoints

The key endpoint GET /api/v1/personal-context/{tenantId}/{userId} is the integration point consumed by Aether Grid's AetherCoreBridgeAgent before every agent decision. This contract must not break without coordinated Grid updates.

MethodPathDescription
GET /api/v1/personal-context/{tenantId}/{userId} Personal context snapshot — primary integration endpoint for Aether Grid
POST /api/v1/users/{userId}/memories Store a new personal memory (type + content + tenantId)
GET /api/v1/users/{userId}/memories/count Total memory count for a user
DELETE /api/v1/users/{userId}/memories/{memoryId} Delete a specific memory
POST /api/v1/sessions Create or resume an active cognitive session
PUT /api/v1/sessions/{sessionId}/turns Append turn summaries, update emotional state and engagement score
POST /api/v1/sessions/{sessionId}/close Close a cognitive session
GET /api/v1/users/{userId}/gdpr/consent Get GDPR consent status
PUT /api/v1/users/{userId}/gdpr/consent Update consent (memory storage, retention period)
DELETE /api/v1/users/{userId}/gdpr/erase GDPR Article 17 — hard-delete all user data (memories, sessions, consent)
GET /actuator/health/liveness Liveness probe — used by Kubernetes and Docker HEALTHCHECK
GET /actuator/health/readiness Readiness probe — signals when Core is ready to serve traffic
